# LINE BOT サーバー 起動手順

---

## ★ 1回で確実に動かす手順（真っ黒を避ける）

何度もやり直しているときは、**Cursor をいったん全部閉じてから**次の手順でやり直すと、うまくいきやすいです。

### 準備: Cursor を閉じる

1. **Cursor を終了する**（ウィンドウを閉じる、または ファイル → 終了）
2. 少し待ってから、**もう一度 Cursor を開く**
3. このプロジェクトのフォルダを開く

### ステップ1: ターミナルを1つだけ開く

4. メニュー **ターミナル** → **新しいターミナル** を選ぶ（または `` Ctrl+` ``）  
   → **ターミナルはこの1つだけ**にする（まだ2つ目は開かない）

### ステップ2: まず ngrok を動かす（1つ目のターミナル）

5. 次のコマンドを**順に**実行する  
   ```bash
   cd "c:\Users\akiko\OneDrive\ドキュメント\AIエンジニア\第５章\5-3領収書読み取りスプシ保存ツール"
   ngrok http 3000
   ```
6. ngrok の画面（https の URL が表示された画面）が**そのまま出ていること**を確認する  
   → **このターミナルにはもう触らない**

### ステップ3: 2つ目のターミナルでサーバーを動かす

7. **2つ目のターミナルを開く**  
   ターミナルパネル内の **＋** ボタン（新しいターミナル）をクリックする  
   または メニュー **ターミナル** → **新しいターミナル**
8. **2つ目のターミナル**で、次のコマンドを実行する  
   ```bash
   cd "c:\Users\akiko\OneDrive\ドキュメント\AIエンジニア\第５章\5-3領収書読み取りスプシ保存ツール"
   npm run start
   ```
9. 「LINE BOT server running on port 3000」と出れば**完了**

### ポイント

- **先に ngrok、後に npm run start** にすると、真っ黒になりにくいことがあります（逆順で試していた場合、この順に変えてみてください）
- ターミナルは**閉じずに**「新しいターミナル」の **＋** で増やす
- 2つとも起動したら、**両方のタブをそのまま**にしておく

### それでも1つ目が真っ黒になる場合

- **1つ目のターミナル**で `npm run start`、**2つ目のターミナル**で `ngrok http 3000` に**逆にして**試す（先に npm run start を実行し、そのあと＋で2つ目を開いて ngrok）
- または **Windows の PowerShell を Cursor の外で開き**、そこで `ngrok http 3000` だけ実行する。Cursor のターミナルでは `npm run start` だけにする（Cursor のターミナル表示の不具合を避けるため）

## 手順1: サーバーを止める（通常の起動し直し）

### 方法A: ターミナルで止める（推奨）

1. サーバーを起動しているターミナルをクリックして選択する
2. キーボードで **`Ctrl + C`** を押す  
   （Mac の場合は **`Control + C`**）
3. ターミナルに `^C` が表示され、プロンプト（`>` や `$`）が戻れば停止完了

### 方法B: ターミナルを閉じる

- そのターミナルタブやウィンドウを閉じると、その中で動いていたサーバーも一緒に止まります

## 手順2: もう一度サーバーを起動する

1. ターミナルでプロジェクトのフォルダに移動する  
   ```bash
   cd "c:\Users\akiko\OneDrive\ドキュメント\AIエンジニア\第５章\5-3領収書読み取りスプシ保存ツール"
   ```
2. 次のコマンドでサーバーを起動する  
   ```bash
   npm run start
   ```
3. 次のメッセージが出れば起動成功です  
   ```text
   LINE BOT server running on port 3000
   ```

## 起動し直したあとにやること

サーバーが起動したら、次を順に確認します。

### 1. ngrok を使っている場合

**重要**: サーバー（`npm run start`）と ngrok は **別々のターミナル** で、**同時に** 動かします。  
同じターミナルで ngrok を実行すると、サーバーが止まったり画面が真っ黒になったりします。

1. **新しいターミナルを1つ開く**（メニュー: ターミナル → 新しいターミナル、または `` Ctrl+Shift+` ``）
2. **サーバーを動かしているターミナルはそのまま**（`npm run start` は止めない）
3. **新しく開いたターミナル**で `ngrok http 3000` を実行する
2. 表示された **https の URL**（例: `https://abc123.ngrok.io`）をコピーする
3. **LINE Developers コンソール**を開く  
   https://developers.line.biz/console/
4. 対象のチャネル → **Messaging API** タブ → **Webhook URL** に  
   `https://あなたのngrokのURL/webhook` を入力して保存する  
   （例: `https://abc123.ngrok.io/webhook`）
5. **「Webhook の利用」** がオンになっているか確認する

### 2. LINE で動作確認する

1. **LINE アプリ**で、このボットを友だち追加（またはトークルームを開く）
2. **テキスト**で領収書の内容を送る  
   （例: `2026年3月5日 電車代 1200円 現金`）  
   → 「領収書を登録しました」などが返ってくれば OK
3. **画像**で領収書の写真を送る  
   → 「登録しました」と日付・店名・科目・金額が返ってくれば OK

### 3. ターミナルを確認する

- サーバーを起動したターミナルに、  
  `受信メッセージ:` や `画像メッセージ受信` などが出ていれば、Webhook は届いています
- **エラー（赤い文字）** が出ていたら、そのメッセージをメモして原因を調べます  
  （例: 環境変数が足りない、スプレッドシートのシート名が違う、など）

## 確認する手順（レシートOCR → スプレッドシート保存）

次の順で確認すると、領収書読み取り〜スプシ保存まで一通りチェックできます。

### 事前準備

1. **サーバーと ngrok を起動**  
   ターミナル1: `npm run start`、ターミナル2: `ngrok http 3000`
2. **Webhook URL を設定**  
   LINE Developers コンソールで、Messaging API の Webhook URL に  
   `https://（ngrokのURL）/webhook` を設定し、「Webhook の利用」をオンにする
3. **スプレッドシートを用意**  
   同じスプレッドシートに「Receipts」という名前のシート（タブ）があることを確認（別名の場合は `.env` の `RECEIPT_SHEET_NAME` を設定）

### 確認ステップ

| 順番 | やること | 確認ポイント |
|------|----------|--------------|
| 1 | **テキストで領収書を送る**  
  LINE で「2025年3月8日 セブン 500円 現金」のように送信 | 「領収書を登録しました」と返ってくるか |
| 2 | **領収書の画像を送る**  
  レシート・領収書の写真を LINE で送信 | 「登録しました」と、日付・店名・勘定科目・支払方法・金額が1行で返ってくるか |
| 3 | **スプレッドシートを開く**  
  設定しているスプレッドシートの「Receipts」シートを開く | 送った領収書の **日付・支払い先・勘定科目・支払方法・金額** が A〜E 列に1行追加されているか |
| 4 | **ターミナルログを見る**  
  `npm run start` を実行しているターミナルを確認 | 「画像メッセージ受信」が出ているか。辞書にない店名の場合は「新店舗候補: ○○」が出るか。エラー（赤文字）が出ていないか |

### うまくいかないとき

- **画像を送っても反応がない**  
  Webhook URL が正しいか、ngrok が起動したままか、サーバーが止まっていないかを確認
- **「領収書を読み取れませんでした」と返る**  
  画像がぼやけていないか、明るさは十分か確認。ターミナルのエラーメッセージを確認
- **スプレッドシートに追記されない**  
  ターミナルに「Unable to parse range」などが出ていないか確認。シート名が「Receipts」（または `.env` の `RECEIPT_SHEET_NAME`）と一致しているか確認

## 手順3: ngrok も使っている場合

LINE の Webhook に **ngrok** を使っているときは、**サーバーと ngrok の両方**を「止める → 起動する」順でやり直します。

### 1. 両方止める

1. **サーバー**のターミナルで **`Ctrl + C`** を押して止める
2. **ngrok** のターミナルでも **`Ctrl + C`** を押して止める

### 2. 起動する順番（2つのターミナルを使う）

**「全部閉じてから新しくターミナルを2つ開いて、1つ目で npm run start、2つ目で ngrok」というやり方でも問題ありません。** その手順が原因で真っ黒になるわけではありません。

| ターミナル | やること | 止めないこと |
|------------|----------|--------------|
| **ターミナル1** | `npm run start` を実行し、**そのまま放置** | サーバーが動いているあいだはここで他のコマンドを打たない |
| **ターミナル2** | `ngrok http 3000` を実行し、**そのまま放置** | ngrok の画面が表示されたままにする |

1. **ターミナル1** で `npm run start` を実行する  
   - 「LINE BOT server running on port 3000」が出たら、**このターミナルには触らない**
2. **ターミナル2** を新規で開き、`ngrok http 3000` を実行する  
   - 表示された **https の URL** を LINE の Webhook URL に設定する  
   - **ターミナル1 のサーバーは止めずに、両方とも起動したまま**にしておく

## よくある失敗

| 状況 | 原因と対処 |
|------|------------|
| **ngrok を実行したら npm run start のターミナルが真っ黒になった** | ① **同じターミナル**で ngrok を打っていないか確認。別ターミナルでやっている場合、② エディタの**描画のずれ**のことがあります。**サーバーのターミナルタブをクリック**して選択し直すと、文字が戻ることがあります。③ それでも真っ黒なら、サーバーが落ちている可能性があるので、**そのターミナルで再度 `npm run start`** を実行し、**もう一方のターミナル**で ngrok を起動し直してください。 |
| `Ctrl + C` を押しても止まらない | ターミナルがサーバー側ではなく別の画面を選択している可能性があります。サーバーが動いているターミナルをクリックしてから、もう一度 `Ctrl + C` を試してください。 |
| 起動しようとしたら「ポートが使われている」と出る | まだ前のサーバーが動いています。タスクマネージャーで `node` を終了するか、該当ターミナルで再度 `Ctrl + C` を押してから `npm run start` を実行してください。 |
| コードを直したのに挙動が変わらない | サーバーを止めずに動かしたままにしていることが多いです。必ず **一度止めてから** `npm run start` で起動し直してください。 |
| **領収書画像で「Unable to parse range: Receipts!A:H」と出る** | スプレッドシートに **領収書用のシート（タブ）が無い**か、**名前が一致していません**。① スプレッドシートを開き、**タブ名が「Receipts」** のシートがあるか確認する。無ければ **新しいシートを追加して名前を「Receipts」に変更**する。② 別の名前のシートを使う場合は、`.env` に **`RECEIPT_SHEET_NAME=実際のタブ名`**（例: `RECEIPT_SHEET_NAME=Sheet1`）を追加して保存し、サーバーを起動し直す。 |

## Google スプレッドシートの準備（領収書画像を使う場合）

領収書画像を送ると、読み取り結果が **同じスプレッドシート内の「領収書用シート」** に追記されます。

- スプレッドシートの **タブ（シート名）** を **「Receipts」** にしたシートを 1 枚用意してください。
- 既存のシート名（例: シート1 / Sheet1）のまま使う場合は、`.env` に次の行を追加します。  
  `RECEIPT_SHEET_NAME=シート1` または `RECEIPT_SHEET_NAME=Sheet1`  
  その後、サーバーを止めてから `npm run start` で再起動してください。

---

## まとめ

1. **止める**: サーバー（と ngrok）のターミナルで **`Ctrl + C`**
2. **起動**: `npm run start`（ngrok を使う場合はそのあとに `ngrok http 3000`）
3. **起動後**: ngrok の URL を Webhook に設定し、LINE でテキストや画像を送って動作確認
4. コードを変えたときは、**毎回この「止める → 起動」をやり直す**
