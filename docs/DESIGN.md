```
│
├── package.json
├── tsconfig.json
└── README.md
```

### 方針

- **types/** … LINE・Sheets・Vision から共通で参照するドメイン型のみを置く。
- **journal/** … 仕訳の「作成・検証・正規化」に専念し、I/O（DB・API）は含めない。
- 将来の **line/** **sheets/** **vision/** は、`journal` をインポートして利用する想定。

## 2. 型定義の設計

### 2.1 勘定区分・勘定科目（account.ts）

- **勘定区分（五勘定）**: 資産・負債・純資産・収益・費用。青色申告・複式簿記の基礎。
- **勘定科目**: 文字列のコード（例: `cash`, `sales`, `expense_office`）と、区分へのマッピングを型で表現。
- 将来の科目マスタ追加や税区分紐づけを考え、区分は enum または union で固定し、科目コードは拡張可能にする。

### 2.2 税区分（tax.ts）

- **今は計算しないがカラムは持つ** 方針。
- 値: `taxable`（課税） / `reduced`（軽減） / `exempt`（免税） / `non_taxable`（不課税） など。
- 仕訳行または伝票単位で「税区分」だけ保持。税率・税額は将来の拡張で追加。

### 2.3 仕訳・仕訳行（journal.ts）

- **1仕訳 = 日付 + 摘要 + 複数行**。各行は「借方 or 貸方」「勘定科目」「金額」「税区分（任意）」を持つ。
- 複式簿記の制約: **借方合計 = 貸方合計**。仕訳エンジンの責務でこれを保証・検証する。

## 3. 仕訳エンジンの責務範囲

### 含めるもの（責務）

| 責務 | 内容 |
|------|------|
| 仕訳の作成 | 日付・摘要・行リストから仕訳オブジェクトを組み立てる |
| 複式の検証 | 借方合計 = 貸方合計のチェック |
| 行の検証 | 金額 > 0、勘定科目の有効性、税区分の有効性 |
| 正規化 | 行の並び順（借方→貸方など）の統一、日付フォーマット |
| 計算 | 合計額の算出（税額計算は将来対応時まで行わない） |

### 含めないもの（非責務）

| 非責務 | 理由 |
|--------|------|
| Google Sheets への読み書き | I/O は sheets モジュールで行う |
| LINE からのメッセージ解釈 | line モジュールの責務 |
| OCR 結果の解釈 | vision モジュールの責務 |
| ユーザー・認証・状態管理 | 別レイヤーの責務 |
| 消費税額の計算 | 将来対応時に対象とする |

### インターフェース方針

- 入力: 日付・摘要・行リスト（借方/貸方・科目・金額・税区分）。
- 出力: 検証済み・正規化済みの仕訳オブジェクト、または検証エラー一覧。
- エンジンは「純粋な関数」に近づけ、同じ入力からは常に同じ出力が得られるようにする（テスト容易性・拡張性）。

---

## 4. 今後の拡張の想定

- **消費税対応**: 税区分に応じた税率マスタと税額計算を `journal` または `tax` に追加。
- **OCR**: Vision が「日付・金額・品目」などを抽出 → `journal` が仕訳オブジェクトを組み立てる。
- **Sheets**: 仕訳オブジェクトをスプレッドシートの行形式に変換する層を `sheets` に実装。
